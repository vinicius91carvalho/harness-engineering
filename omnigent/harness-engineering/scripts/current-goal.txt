Turn causeflow-ai (a monorepo of core, web, relay, public-docs) fully open
source and Docker-runnable. Remove AWS, Stripe, Clerk, and other paid/
external SaaS dependencies entirely from this subproject. Keep only what is
essential to the product working (e.g. Hindsight). The result should
`docker compose up` and run without any of the removed vendors configured --
so any AWS-backed persistence/queue/credentials/dispatch, any Stripe billing,
and any Clerk auth needs a self-hosted/local equivalent (e.g. Postgres for
DynamoDB, a local queue for SQS, local env-based secrets for STS/KMS, local
JWT auth for Clerk, a stub/free-tier billing service for Stripe). Add these
as new Acceptance Checks (depends_on the relevant existing checks where
appropriate) -- do not remove or rewrite existing checks. Match whatever
approach the other subprojects in this monorepo are already taking for the
same vendor (check sibling `project_specs.xml` files if present) so the
choices are consistent across the whole workspace.
