# Fleet ops automation waits on Fleet Snapshot

Cross-project monorepo recovery (governor contention, sibling bearings, fleet-wide diagnose) must consume a structured Fleet Snapshot interface first, not absorb `monorepo-supervisor-ops` narrative into `harness-control.mjs`.
Single-project automations that already have local helpers (finished-tab reap, spawn-silence health, evidence-bound retry guidance) may deepen without waiting.
Wholesale skill dump into the Control Host would fight ADR-0007 and ADR-0009.
