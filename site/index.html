Hermes + Telegram + OpenCode

One path from idea to verified code.

Hermes receives the goal and reports through Telegram. OpenCode runs the workers. The harness owns planning, parallel work, QA, retries, integration, and the final Goal Review.

“It's the job that's never started as takes longest to finish.”

Before you start

What runs where

TelegramYour remote chat and notification channel.
HermesThe always-on control host that receives goals and relays events.
HarnessThe durable scheduler, state machine, QA loop, and evidence store.
OpenCodeThe authenticated worker CLI that edits and tests the project.

Run all commands on the same Linux, macOS, or WSL machine that has the target Git repository. You need Git, Bash, Node.js, jq, and an LLM provider for both Hermes and OpenCode.

Step 1

Install Hermes and OpenCode

Install both CLIs, reload your shell, then configure one model provider in each.

curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
curl -fsSL https://opencode.ai/install | bash

source ~/.zshrc  # use ~/.bashrc when running Bash
hermes setup
opencode         # then run /connect and exit
Checkhermes doctoropencode --version

Created: ~/.hermes/config.yaml, ~/.hermes/.env, and OpenCode's user configuration.

Step 2

Install the harness for OpenCode

The non-interactive command installs only the harness and targets OpenCode explicitly.

curl -sSL https://raw.githubusercontent.com/vinicius91carvalho/harness-engineering/main/install.sh \
  | sh -s -- --cli opencode --no
Checktest -f ~/.config/opencode/skills/harness-planner/SKILL.mdtest -f ~/.config/opencode/commands/harness-generator.md

Created: namespaced skills, agents, and commands under ~/.config/opencode/.

Step 3

Give Hermes the control skills

Keep one checkout as the source of truth and symlink the three sibling skills. Their relative script paths must stay together.

git clone https://github.com/vinicius91carvalho/harness-engineering.git \
  ~/harness-engineering
mkdir -p ~/.hermes/skills
ln -s ~/harness-engineering/skills/control-host ~/.hermes/skills/control-host
ln -s ~/harness-engineering/skills/planner ~/.hermes/skills/planner
ln -s ~/harness-engineering/skills/generator ~/.hermes/skills/generator

hermes skills list

Created: three links under ~/.hermes/skills/. The scripts remain in ~/harness-engineering/skills/.

Step 4

Connect Hermes to Telegram

  1. In Telegram, open @BotFather, run /newbot, and copy the bot token.
  2. Run the gateway wizard, select Telegram, and paste the token.
  3. Install and start the gateway service, then message your bot.
  4. If Hermes returns a pairing code, approve it locally.
hermes gateway setup
hermes gateway install
hermes gateway start
hermes gateway status

# Only when the bot sends a pairing code:
hermes pairing approve telegram YOUR_PAIRING_CODE

Security: do not enable public access. Pair your own account or set TELEGRAM_ALLOWED_USERS in ~/.hermes/.env.

Created: Telegram credentials in ~/.hermes/.env, gateway settings, and a user service.

Step 5

Plan the project

Start OpenCode in the target repository. Invoke the namespaced planner command and describe observable user outcomes.

cd /path/to/your-project
opencode

/harness-planner Build a notes app where a user can create a note,
reload the page, and still find it.

Review the generated checks before continuing. Each check should describe behavior that QA can actually observe.

Created: project_specs.xml — the goal, design decisions, and stable Acceptance Checks.

Step 6

Create the execution queue

In the same OpenCode session, invoke the generator once. It initializes Git state, maps checks into Work Items, and can start the first worker.

/harness-generator

Created: feature_list.json, CONTEXT.md, harness-progress/, and Run State under .git/harness-runs/.

Step 7

Hand the long run to Hermes

From Telegram, send Hermes a concrete instruction. The skill inspects existing state, starts the detached supervisor, and uses OpenCode for worker processes.

/control-host
Run the harness for /path/to/your-project with OpenCode workers.
Use at most 2 workers. Report progress and blockers in this chat.

Hermes should run the equivalent of:

CONTROL=~/harness-engineering/skills/control-host/scripts/harness-control.mjs
node "$CONTROL" start --repo /path/to/your-project --host opencode \
  --max-workers 2 --quota-workers 2
node "$CONTROL" status --repo /path/to/your-project

Created: supervisor state and events under .git/harness-control/; per-context branches, worktrees, journals, and evidence as work proceeds.

Step 8

Keep Telegram synchronized

The control-host skill tells Hermes to poll durable events at least once per minute, deliver them, then acknowledge them. Ask Hermes to create that recurring job:

Poll harness events for /path/to/your-project every minute using the
control-host skill. Deliver progress and input requests here. Acknowledge
an event only after it has been delivered.

For direct inspection, these are the underlying commands:

node "$CONTROL" events --repo /path/to/your-project \
  --consumer hermes-telegram
node "$CONTROL" ack --repo /path/to/your-project \
  --consumer hermes-telegram --event EVENT_ID

Created: the Hermes cron entry in ~/.hermes/cron/jobs.json and the delivery cursor in .git/harness-control/cursors/hermes-telegram.json.

Step 9

Verify the whole chain

1
Gateway is onlinehermes gateway status
2
OpenCode is the workernode "$CONTROL" status --repo /path/to/your-project
3
The plan is validnode ~/harness-engineering/skills/generator/reconcile.mjs /path/to/your-project --check
4
Telegram receives eventsnode "$CONTROL" events --repo /path/to/your-project --consumer hermes-telegram
5
The goal actually passed

Wait for a durable run_completed event with goal:true. An empty queue or a confident agent message is not completion.

Operations

Inspect, pause, resume, or stop

node "$CONTROL" status --repo /path/to/your-project
node "$CONTROL" capacity --repo /path/to/your-project --host opencode
node "$CONTROL" pause  --repo /path/to/your-project
node "$CONTROL" resume --repo /path/to/your-project
node "$CONTROL" stop   --repo /path/to/your-project

If a worker blocks, reply in Telegram with guidance tied to the Input Request event ID. The harness preserves the branch, worktree, defect report, repair plan, and evidence.
